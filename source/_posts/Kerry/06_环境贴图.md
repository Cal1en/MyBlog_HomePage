# 立方体纹理的局限性
使用立方体贴图作为环境反射的采样方式具有局限性：我们如果只根据方向来采样`CubeMap`的话，意味着如果几何体上有两个点的反射方向相同，那么他们会采样到同一个点，尽管这两个点的物理间距可能很大。

因此，`CubeMap`技术不适合使用在平面模型上。 

# 如何在移动端使用HDR贴图
部分移动端无法使用PC端所使用的`BC6H`的HDR贴图压缩方式，只能使用`RGBM`压缩方式，其原理是将HDR贴图的亮度存入A通道中。

如果我们要在移动端使用`BC6H`的HDR贴图，我们就需要在`Shader`中额外声明一个`float4`类型的变量，其变量名为`立方体贴图名_HDR`（`立方体贴图名`表示使用了HDR的`samplerCUBE`），然后调用`DecodeHDR()`函数进行解码：

```csharp
samplerCUBE _CubeMap;
float4 _CubeMap_HDR;
/...
    half4 color_cubemap = texCUBE(_CubeMap, reflect_dir);
    color_cubemap.xyz = DecodeHDR(color_cubemap, _CubeMap_HDR);
```

# 旋转采样立方体贴图
```csharp
Shader "Unlit/CubeMap_code"
{
    Properties
    {
        _MainTex ("Texture", 2D) = "white" {}
        _CubeMap("CubeMap", Cube) = ""{}
        _Rotate("Rotate", Range(0,360)) = 0
    }
    SubShader
    {
        Tags { "RenderType"="Opaque" }
        LOD 100

        Pass
        {
            CGPROGRAM

            #include "Lighting.cginc"

            #pragma vertex vert
            #pragma fragment frag

            struct v2f
            {
                float2 uv : TEXCOORD0;
                float4 vertex : SV_POSITION;
                float3 reflectDir : TEXCOORD1;
            };

            sampler2D _MainTex;
            float4 _MainTex_ST;
            samplerCUBE _CubeMap;
            float _Rotate;

            v2f vert (appdata_base v)
            {
                v2f o; 
                o.vertex = UnityObjectToClipPos(v.vertex);
                o.uv = TRANSFORM_TEX(v.texcoord, _MainTex);
                half3 worldPos = mul(unity_ObjectToWorld, v.vertex);
                half3 worldNormal = normalize(UnityObjectToWorldNormal(v.normal));
                o.reflectDir = reflect(-normalize(UnityWorldSpaceViewDir(worldPos)), worldNormal);
                half s,c;
                sincos(radians(_Rotate), s, c);
                half2x2 rotate = half2x2(c, -s, s, c);
                o.reflectDir.xz = mul(rotate, o.reflectDir.xz);
                return o;
            }

            fixed4 frag (v2f i) : SV_Target
            {
                fixed4 col = texCUBE(_CubeMap, normalize(i.reflectDir));
                return col;
            }
            ENDCG
        }
    }
}

```

# 反射探针
我们可以在场景中创建反射探针，并通过反射探针烘焙环境立方体贴图，生成的立方体贴图文件会保存到项目文件夹中（或者我们可以将反射探针的类型改为`自定义`，即可手动更改保存位置）。

我们可以使用`Unity Shader`自带的变量`unity_SpecCube0`和`unity_SpecCube1`来获取反射探针生成的立方体贴图，我们可以观察到反射探针在场景中有一个立方体包围盒，该包围盒的作用是，**包围盒内的物体在`Shader`中调用`unity_SpecCube0`即可获得该反射探针生成的立方体贴图**。我们需要使用以下函数来采样`unity_SpecCube0`：

`float4 rgbm = UNITY_SAMPLE_TEXCUBE(unity_SpecCube0, reflectDir);`

该函数同时会自动调用`DecodeHDR()`对立方体贴图的HDR信息进行解码。

+ `unity_SpecCube1`的作用：如果当前物体同时位于两个反射探针的包围盒中，那么**Unity会根据两个反射探针的**`Importance`**属性值大小进行排序，重要性最大的存入**`unity_SpecCube0`**，第二高的存入**`unity_SpecCube1`**；**
+ 如果当前物体正在从一个反射探针的包围盒范围进入另一个反射探针的包围盒，我们可以在`unity_SpecCube0`和`unity_SpecCube1`之间进行线性插值来计算结果。
+ 如果我们希望**反射探针能够采集周围的其他物体**，我们需要将这些物体的`static`设置为`Reflection Probe Static`。
+ **Unity会自动在场景中放置一个全局的反射探针**，我们只需要在`Lighting`窗口中点击`生成光照`，Unity就会生成该立方体贴图，并存入`unity_SpecCube0`中（如果当前物体不在任何其他反射探针的包围盒中），我们可以在`Lighting - Environment`窗口中的环境反射面板调整相关设置。
